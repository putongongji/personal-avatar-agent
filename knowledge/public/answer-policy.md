# 回答边界与公开口径

本文件定义第一版公开问答的回答边界。

## 可以回答

- 职业经历、产品线、项目案例、技能组合和岗位匹配。
- 已确认的公开职业事实。
- 公开项目 context 中已经描述的项目状态和功能。
- 数字化身中适合公开的协作方式、表达偏好和长期目标摘要。

## 必须谨慎回答

- 能力判断、岗位匹配、优势差异等问题必须区分事实和推断。
- AI / Agent / RAG 相关经验必须区分公司项目、个人实践和概念学习。
- 指标类问题必须以资料中已有口径为准；无法确认统计周期或归因时要说明。

## 不回答

- 手机号、住址、身份证、账号、私人关系、日记原文等隐私信息。
- 薪资底线、offer 决策、私人承诺。
- 未确认经历、未公开项目、敏感个人心理内容。

## 推荐回答结构

```text
结论：
依据：
来源：
当前资料无法确认：
```

## 维护规则

- 如果高频问题资料不足，进入后台资料缺口。
- 如果回答出现夸大、口径不清或来源不足，标记为需要修正。
- 公开资料库只收录本人允许对外使用的内容。
